---

## Память проекта — `STATE.md`

В этом репозитории живёт файл `STATE.md` — единая память о проекте, которая переживает все сессии и доступна любому чату Claude (Cowork, Sonnet code chat, Claude Code, claude.ai). Это "облачная память" через git: как только она закоммичена и запушена, любая будущая сессия читает её и сразу знает текущее состояние.

### Жёсткие правила

1. **Старт сессии — первым делом читаю `STATE.md`.** Если его нет — создаю по шаблону. Если есть — опираюсь как на свежайшее знание о проекте.

2. **После каждого значимого изменения** (commit, deploy, правка env, найденный баг, принятое решение, миграция, новая фича) — **обновляю `STATE.md` и коммичу его в одном коммите с правкой**.
   - Запись в "Последние изменения" сверху (новые впереди).
   - Обновляю "Текущее состояние", "Известные проблемы", "Следующие шаги" если общая картина поменялась.

3. **Не пишу в `STATE.md`:** секреты (токены, пароли, ключи) — только ИМЕНА env-переменных. Персональные данные. Длинные диффы — ссылки на коммиты.

4. **Стиль:** markdown, компактно, без воды. Дата+время `YYYY-MM-DD HH:MM`.

5. **Если STATE.md противоречит коду** — приоритет у кода, обновляю STATE.md.

### Краткое правило

«Открыл проект → прочитал `STATE.md`. Что-то сделал → дописал сверху → закоммитил вместе с правкой → запушил.»
