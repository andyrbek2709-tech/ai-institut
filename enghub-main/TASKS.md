## Приоритет 1 — БЛОКЕРЫ (без них система не работает)

### T1. Триггер `task_history` сломан (B1)

- **Симптом:** При INSERT в `tasks` Supabase возвращает `column "field" of relation "task_history" does not exist`.
- **Причина:** Триггер на `tasks` обращается к колонке `task_history.field`, которой больше нет (переименована/удалена).
- **Где чинить:** Supabase SQL editor (прямой доступ к БД, не код фронта).
- **Что сделать:**
  - Зайти в Supabase Dashboard → SQL Editor.
  - `SELECT pg_get_triggerdef(oid) FROM pg_trigger WHERE tgrelid = 'tasks'::regclass;` — посмотреть текст триггера.
  - `\d task_history` (или `SELECT column_name FROM information_schema.columns WHERE table_name='task_history';`) — посмотреть актуальные колонки.
  - Решить: либо `ALTER TABLE task_history ADD COLUMN field text;` (если поле должно быть), либо переписать триггер на актуальные колонки, либо `DROP TRIGGER ... ON tasks;` (если task_history больше не нужен).
- **Без этого ничего из остального тестировать невозможно** — задачи не создаются.

### T2. Проверить рассинхрон полей `title/assignee_id` ↔ `name/assigned_to` (B2)

- **Симптом по отчёту:** Фронт шлёт `{title, assignee_id}`, БД ждёт `{name, assigned_to}` → 400.
- **В текущем коде** (`src/App.tsx`, функция `createTask`, ~строка 950) фронт уже шлёт `name` и `assigned_to`. То же в `src/api/supabase.ts` → `createProjectTask`.
- **Что сделать:**
  - `grep -rn "title:\|assignee_id" src/` по всему `enghub-main/src/` — убедиться, что нигде не осталось старых ключей.
  - Если grep чистый — это **фантомный баг**: на проде стоит старый билд. Закрыть после деплоя свежей сборки.
  - Если grep что-то найдёт — заменить на `name`/`assigned_to`.

### T3. RLS на таблице `tasks` отсутствует (B4)

- **Симптом:** Залогиненный пользователь через прямой Supabase REST читает задачи всех проектов (включая чужие).
- **Сначала верифицировать:** убедиться, что тестировщик ходил с `apikey: anon`, а не `service_role` (последний легально обходит RLS).
- **Где чинить:** Supabase → Authentication → Policies → таблица `tasks`.
- **Что сделать (минимум):**
  ```sql
  ALTER TABLE tasks ENABLE ROW LEVEL SECURITY;

  -- Чтение: только если задача в проекте, к которому у пользователя есть доступ
  CREATE POLICY tasks_select ON tasks FOR SELECT
    USING (
      project_id IN (
        SELECT p.id FROM projects p
        WHERE p.gip_id = auth.uid()
           OR EXISTS (SELECT 1 FROM app_users u WHERE u.supabase_uid = auth.uid() AND u.role = 'gip')
           OR EXISTS (SELECT 1 FROM project_depts pd JOIN app_users u ON u.dept_id = pd.dept_id
                      WHERE pd.project_id = p.id AND u.supabase_uid = auth.uid())
      )
    );

  -- Вставка/обновление/удаление — по аналогии (роль-зависимо)
  ```
- **Уточнить с владельцем:** какая именно бизнес-логика доступа (GIP видит свои проекты? lead видит проекты своего отдела? engineer — только назначенные ему задачи?). Без этого политики писать наугад опасно.

### T4. LiveKit (видеовстречи) сломаны (B3)

- **Симптом:** Вкладка "Совещание" → сразу "Не удалось создать или найти встречу". Сетевых запросов к LiveKit нет.
- **Гипотеза:** падает раньше fetch, на проверке env-переменной или серверной функции, выдающей JWT.
- **Где смотреть:**
  - `enghub-main/api/` (Vercel serverless functions) — найти endpoint, выдающий LiveKit JWT/URL.
  - Логи последнего деплоя в Vercel Dashboard → Functions → Logs.
  - Переменные окружения в Vercel: `LIVEKIT_API_KEY`, `LIVEKIT_API_SECRET`, `LIVEKIT_URL` (или `REACT_APP_LIVEKIT_URL` для фронта).
- **Что сделать:** запросить лог одного неудачного клика "Совещание", найти стектрейс, починить корневую причину. Если ключи протухли — обновить.

### T8. Multiple GoTrueClient instances (B11)

- **Симптом:** в консоли 4+ предупреждения `Multiple GoTrueClient instances detected`.
- **Причина:** `createClient()` от Supabase вызывается в нескольких местах без синглтона.
- **Где искать:** `grep -rn "createClient" src/`. В `App.tsx` строка ~619 — `createClient` для realtime. Возможно ещё в других компонентах.
- **Что сделать:** вынести в `src/api/supabase.ts` единый экспортируемый клиент:
  ```ts
  import { createClient } from '@supabase/supabase-js';
  let _client: any = null;
  export const getSupabase = () => {
    if (!_client) _client = createClient(SURL, KEY);
    return _client;
  };
  ```
  И заменить все локальные `createClient(...)` на `getSupabase()`.
