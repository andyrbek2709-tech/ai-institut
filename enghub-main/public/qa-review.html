Корневая причина (подтверждено SQL)

На таблице public.tasks висят три триггера. Один корректный (использует field_name), два сломанных (используют field + comment, которых в таблице нет):

Колонки task_history: id, task_id, changed_by, field_name, old_value, new_value, changed_at → нет ни «field», ни «comment» Триггер task_changes_trigger (UPDATE) — OK: INSERT INTO task_history(task_id, field_name, old_value, new_value)… Триггер trg_tasks_history (UPDATE) — СЛОМАН: INSERT INTO task_history(task_id, field, old_value, new_value, changed_by, comment)… Триггер trg_tasks_history_create (INSERT) — СЛОМАН: INSERT INTO task_history(task_id, field, old_value, new_value, changed_by) VALUES (new.id, 'created', null, new.status, who); → срабатывает на КАЖДЫЙ INSERT → блокирует создание любой задачи

Решение (1 SQL-команда)

DROP TRIGGER IF EXISTS trg_tasks_history ON public.tasks; DROP TRIGGER IF EXISTS trg_tasks_history_create ON public.tasks; -- task_changes_trigger остаётся, он корректный и логирует UPDATE. -- Потерь функциональности нет: рабочий триггер уже пишет в task_history.

Время фикса: 30 секунд через Supabase MCP. Можно сделать прямо сейчас.

Что реально на сервере

RLS включён, политик 5: tasks_select (READ) USING auth_is_admin_or_gip() OR … tasks_write_gip (ALL) USING auth_is_admin_or_gip() tasks_write_lead (INSERT) CHECK lead+своего dept tasks_update_lead (UPDATE) USING lead+своего dept tasks_update_engineer (UPDATE) USING engineer+assigned_to=я auth_is_admin_or_gip(): return role IN ('admin','gip') // без фильтрации по проектам // Любой ГИП видит задачи ВСЕХ проектов, не только своих. // Тестировщик это и поймал.

Это баг или фича?

Зависит от того, сколько в системе ГИПов и как они работают между собой. Варианты:

А) Если ГИП один и видит всё — текущая логика корректна, тестировщик ошибся в трактовке.

Б) Если ГИПов несколько и у каждого свой портфель проектов — текущая политика дырявая, нужно добавить фильтр project_id IN (SELECT id FROM projects WHERE gip_id = auth_app_user_id()).

В) Если ГИП-руководитель должен видеть всех, а ГИП-проектировщик — только свои — нужно ввести подроль (gip_lead vs gip_pm).
