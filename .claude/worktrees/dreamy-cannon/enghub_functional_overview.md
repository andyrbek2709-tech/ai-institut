# Обзор системы: EngHub AI Orchestrator

Этот документ описывает бизнес-логику, цели и функциональные возможности платформы EngHub.

## 1. Концепция и цели
**EngHub** — это единая цифровая среда для управления инжиниринговыми проектами на всех этапах проектирования. 

**Главные цели**:
- Исключить потерю данных при передаче заданий между отделами.
- Автоматизировать рутинные инженерные расчеты.
- Обеспечить быстрый поиск по нормативной базе с помощью ИИ (RAG).
- Повысить прозрачность работы ГИПа над сложными многоэтапными проектами.

---

## 2. Основной функционал

### А. Управление проектами и задачами
Система работает по каскадной модели:
1. **ГИП** создает проект и назначает задачи или "Межотдельские задания" ведущим инженерам (Lead).
2. **Lead** распределяет эти задачи внутри своего отдела обычным инженерам.
3. **Инженер** выполняет работу и меняет статус на "Review".
4. **Lead/ГИП** проверяют работу: либо принимают (Done), либо отправляют на доработку (Revision).

### Б. Система ревизий (R0, R1, R2...)
Это уникальная фича EngHub. Когда ГИП выпускает новую ревизию (например, изменился техпроцесс): 
- Система позволяет клонировать существующие задачи в новую ревизию.
- Все замечания и история из R0 переносятся в R1, чтобы инженер видел, что именно изменилось.

### В. Инженерные расчеты (Calc Engine)
В платформу встроено более 90 шаблонов расчетов:
- Выбор кабеля, расчет теплопотерь, материальный баланс и др.
- Результаты расчетов сохраняются в базе данных и могут быть в один клик экспортированы в отчет format `.docx`.

### Г. База знаний и AI Copilot (Phase 7)
- **Нормативка**: Централизованное хранилище ГОСТов, ОСТов и внутренних стандартов.
- **AI Copilot**: Чат-бот, который не просто общается, а умеет искать ответы в загруженных документах (RAG). Инженер может спросить: "Какие требования к факельным системам по ГОСТ 21.101?", и ИИ найдет ответ в базе.

## 3. Ролевые взаимосвязи

| Роль | Основные действия в системе |
| :--- | :--- |
| **ГИП (GIP)** | Создание проектов, выпуск ревизий, глобальный мониторинг, финальная приемка. |
| **Нач. отдела (Lead)** | Контроль нагрузки своих инженеров, верификация расчетов, прием заданий от других отделов. |
| **Инженер** | Проведение расчетов, отчетность по задачам, работа с базой знаний. |
| **Admin** | Настройка оргструктуры, управление доступами. |

---

## 4. Логические потоки данных (Data Flows)

1. **Flow "Задание смежнику"**:
   - Отдел ТХ (Технологи) формирует задание для отдела Э (Электрики).
   - Спецификация оборудования попадает в "Inbox" нач. отдела Э.
   - Пока нач. отдела Э не нажмет "Accept", ГИП видит задержку в матрице увязки.

2. **Flow "Расчет -> Отчет"**:
   - Инженер вводит данные в форму расчета.
   - Движок вычисляет значения.
   - Функция `DocxExporter` собирает эти данные в шаблон и отдает готовый файл для печати.

3. **Flow "RAG Search"**:
   - Пользователь загружает PDF.
   - (Будущее) n8n векторизирует файл.
   - Пользователь пишет вопрос в Copilot.
   - Сервер ищет схожие фрагменты текста и формирует ответ.

---

## 5. Итог
EngHub превращает хаос из Excel-таблиц и почтовых переписок в структурированную базу данных, где каждая цифра и каждое решение подкреплены документально.
