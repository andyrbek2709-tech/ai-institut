### 🔥 TX, TT — Тепловые системы (2 расчёта)

| Категория | Расчёты | Справочный документ | Стандарты |
|-----------|---------|-------------------|-----------|
| **TX** | `tx_heat_balance` — Тепловая мощность | **THERMAL_METHODS_STANDARDS.md** (раздел 1) | СП 60.13330.2020, СП 50.13330.2012, ГОСТ 30494, ISO 13789 |
| **TT** | `tt_pressure_drop` — Потери давления | **THERMAL_METHODS_STANDARDS.md** (раздел 2) | СНиП 2.04.01-85, СНиП 2.04.02-84, СП 30.13330.2020, ISO 5241 |

**Дополнительные ресурсы:**
- 📄 **THERMAL_CALCULATIONS_QUICK_GUIDE.md** — практический гайд с 3 примерами, шпаргалки, частые ошибки
- 📄 **THERMAL_CALCULATIONS_INDEX.md** — навигация и быстрые ответы

**Формулы:**
```
tx_heat_balance:    Q = G · c · ΔT  [кВт], где G — расход [кг/с], c — удельная теплоёмкость, ΔT — разность температур [°C]
tt_pressure_drop:   ΔP = λ · (L/d) · (ρ · v² / 2)  [Па], где λ — коэф. трения, L — длина, d — диаметр, ρ — плотность, v — скорость
```

**Таблицы данных:**
- Теплоёмкость сред: вода 4.18, воздух 1.0, масло 2.0 кДж/(кг·°C)
- Шероховатость труб: ПЭ 0.005 мм, сталь б/у 0.3 мм, чугун 1.5 мм
- Местные сопротивления: колено 1.0, клапан 3.0, фильтр 5.0

## БЫСТРАЯ НАВИГАЦИЯ

### Мне нужно...

#### ...рассчитать тепловую мощность котла

→ **Документ:** THERMAL_METHODS_STANDARDS.md, раздел 1  
→ **Формула:** Q = G · c · ΔT  
→ **Пример:** Дом 200 м² требует 25–30 кВт  
→ **Расчёт в платформе:** `tx_heat_balance`

#### ...определить потери давления в трубе

→ **Документ:** THERMAL_METHODS_STANDARDS.md, раздел 2  
→ **Формула:** ΔP = λ · (L/d) · (ρ · v² / 2)  
→ **Пример:** Стальная труба 50 м, диаметр 50 мм, v = 2 м/с → ΔP ≈ 251 кПа  
→ **Расчёт в платформе:** `tt_pressure_drop`

#### ...рассчитать объём выемки земляных работ

→ **Документ:** GEODESIC_STANDARDS_AND_METHODOLOGIES.md, раздел 2  
→ **Методы:** срезы, треугольники, призмы  
→ **Пример:** Котлован 50×30×2 м = 3000 м³  
→ **Расчёт в платформе:** `g_ob_em_vyemki_nasypi`

#### ...определить баланс земляных масс

→ **Документ:** GEODESIC_STANDARDS_AND_METHODOLOGIES.md, раздел 3 + **SOIL_LOOSENING_COEFFICIENTS_REFERENCE.md**  
→ **Формула:** ΔV = V_выемка·K_разр - V_насыпь/K_уплотн  
→ **Таблица K:** 50+ типов грунтов (гранит 1.5–1.7, суглинок 1.1–1.2, песок 1.05–1.12)  
→ **Пример:** Выемка 3000 м³, насыпь 1200 м³, суглинок → прибыль 892 м³  
→ **Расчёт в платформе:** `g_balans_zemlyanyh_mass`

#### ...привязать объект по координатам

→ **Документ:** GEODESIC_STANDARDS_AND_METHODOLOGIES.md, раздел 4  
→ **Методы:** полярный (расстояние + азимут), ортогональный (по базису), GPS  
→ **Трансформация:** СК-95 ↔ WGS84  
→ **Точность:** ±0.05–0.1 м (теодолит), ±1 м (GPS дифференциальный)  
→ **Расчёт в платформе:** `g_raschet_koordinat`, `g_privyazka_ob_ektov`

#### ...найти стандарт для моего расчёта

→ **Документ:** Этот файл (CALCULATIONS_STANDARDS_INDEX.md)  
→ **Таблица:** "КАТЕГОРИИ РАСЧЁТОВ И СПРАВОЧНЫЕ ДОКУМЕНТЫ" выше  
→ **По категории:** TX, TH, EO, VK, G, PB

## ИНТЕГРАЦИЯ В FOUNDATION PHASE

**Foundation Phase** (недели 1–3, 50 часов):

1. **Backend (FastAPI)** — создание `CalculationEngine` с:
   - Загрузка шаблонов из справочников
   - Валидация против стандартов
   - Вычисление по формулам
   - Логирование (для аудита)

2. **Frontend (React)** — отображение:
   - Входные параметры (с hints из справочника)
   - Формулу (с LaTeX из справочника)
   - Результаты
   - Нормативные ссылки

3. **Database (Supabase)** — хранение:
   - `calculation_templates` — данные из справочников
   - `calculation_results` — результаты пользователей
   - `calculation_audit` — для аудита
