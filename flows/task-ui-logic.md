## 1. МАКЕТ ПРИЛОЖЕНИЯ

### Общая структура

```
┌─────────────────────────────────────┐
│  EngHub — Инженерная платформа      │
├─────────────────────────────────────┤
│  [Логотип] | Мой список | Очередь | Dashboard | [Профиль ▼]
├─────────────────────────────────────┤
│                                     │
│  ОСНОВНОЙ КОНТЕНТ                   │
│  (зависит от роли и выбранного      │
│   экрана)                           │
│                                     │
│                                     │
└─────────────────────────────────────┘
```

### Навигация (верхний бар)

| Элемент | Видно | Переходит | Примечание |
|---|---|---|---|
| **Мой список** | Инженер, Lead, ГИП | Список МОИХ задач | У каждой роли свой список |
| **Очередь** | Lead | Список задач в статусе "На проверке Lead" | Отдела Lead |
| **Очередь ГИПа** | ГИП | Список задач в статусе "На проверке ГИП" | Проекта ГИПа |
| **Dashboard** | ГИП | Аналитика проекта | Только ГИП видит |
| **Профиль** | Все | Профиль, выход | Текущий пользователь |

│                                            │

```

### Вид КАНБАН

**Колонки:**
1. **СОЗДАНА** — статус `created`
2. **В РАБОТЕ** — статус `in_progress`
3. **НА ПРОВЕРКЕ** — статус `review_lead`
4. **НА ДОРАБОТКЕ** — статус `rework`
5. **ОЖИДАНИЕ ДАННЫХ** — статус `awaiting_data` (блокированные задачи)

**Карточка задачи в канбане:**

```
┌─────────────────────────┐
│ 🔴 КЖ-2026-Fase-1_Схема │ ← Красный кружок = блокирована
│                         │
│ Дедлайн: 7 мая          │
│                         │
│ 🚨 ТРЕБУЕТСЯ ЭС         │ ← Красная иконка = нужна другим
│                         │
│ Ответственный: Гриценко │
└─────────────────────────┘
```

**Индикаторы на карточке:**

| Индикатор | Значение | Как показывается |
|---|---|---|
| 🔴 | Блокирована (ожидание данных) | Красный кружок слева |
| 🚨 | Требуется другим отделам | Красная иконка "alert" |
| ⏱️ | Срочно (дедлайн < 2 дня) | Оранжевые часы |
| ✅ | На проверке | Синий чекбокс |
| ❌ | Возвращена (замечания) | Красный X, количество |
| 📎 | Есть файлы | Скрепка рядом с числом |

### Вид ТАБЛИЦА

```
┌────────────────────────────────────────────────────────────────┐
│ Задача         │ Статус  │ Дедлайн    │ Замечания │ Файлы  │ │
├────────────────────────────────────────────────────────────────┤
│ КЖ-2026-Fase-1 │ 🔴 БЛК  │ 7 мая      │ -         │ 2 📎   │ │
│ ЭС-2026-Расчё  │ ✏️ РАБОТ │ 10 мая     │ -         │ 1 📎   │ │
│ АС-2026-Чертёж │ ✅ ПРОВ │ 8 мая      │ 2 ❌      │ 3 📎   │ │
└────────────────────────────────────────────────────────────────┘
```

### Фильтры

**Доступные фильтры:**
- По статусу (мультивыбор)
- По дедлайну (< 1 дня, < 1 недели, < 1 месяца)
- По проекту (мультивыбор, если инженер в несколько проектов)
- По наличию замечаний (да/нет)

│                                              │

│                                              │
└──────────────────────────────────────────────┘
```

### Структура карточки в очереди

| Элемент | Показывается | Функция |
|---|---|---|
| Название задачи | Всегда | Кликабельное → открыть карточку |
| Инженер | Всегда | Информационное |
| Время на проверке | Всегда | Показать, сколько уже ждём |
| Дедлайн | Всегда | Красный если < 24 часов |
| Кол-во замечаний ранее | Если > 0 | Жёлтый текст "Замечания ранее: N" |
| Кнопка [ОТКРЫТЬ] | Всегда | Переход в карточку |
| Кнопка [✅ ПРИНЯТЬ] | Если нет блокирующих замечаний | Отправить ГИПу |
| Кнопка [❌ ВЕРНУТЬ] | Всегда | Вернуть на доработку |

### Статистика в хедере

```
Всего на проверке: 5
├─ Не просмотрены: 2
├─ Просмотрены, есть замечания: 2
└─ Готовы к отправке: 1
```

## 10. СТАТУСЫ В UI: ЦВЕТА И ИНДИКАТОРЫ

### СТАТУСЫ ЗАДАЧИ

| Статус | Код | Цвет | Иконка | Как показывается | Описание |
|---|---|---|---|---|---|
| **Создана** | `created` | 🟡 Жёлтый | 📝 | "Создана" | ГИП создал, инженер ещё не начал |
| **В работе** | `in_progress` | 🟢 Зелёный | ✏️ | "В работе" | Инженер активно работает |
| **На проверке Lead** | `review_lead` | 🔵 Синий | 👁️ | "На проверке" | Lead проверяет результаты |
| **На проверке ГИП** | `review_gip` | 🔵 Синий | 🏛️ | "На проверке у ГИПа" | ГИП делает финальную проверку |

| **Ожидание данных** | `awaiting_data` | 🔴 Красный | ⏳ | "Блокирована" | Ждёт данных от другой задачи |
| **Утверждено** | `approved` | 🟢 Зелёный (тёмный) | ✅ | "Утверждено" | ГИП одобрил, готово к выпуску |

### ИНДИКАТОРЫ И ЗНАЧКИ

| Индикатор | Значение | Когда показывается | Где видно |
|---|---|---|---|
| 🔴 Красный круг | Блокирована | Статус = awaiting_data | Канбан, таблица, список |
| 🚨 Красный alert | Требуется другим | Есть задача, которая зависит от этой | Канбан, таблица |
| ⏱️ Оранжевые часы | СРОЧНО | Дедлайн < 24 часа | Канбан, таблица, карточка |
| ❌ Красный X | Возвращена | Замечания не разрешены | Канбан, таблица, список |
| 📎 Скрепка | Файлы | Есть загруженные файлы | Канбан, таблица, карточка |
| 💬 Облако | Замечания | Есть замечания | Канбан, таблица, карточка |
| 🔗 Цепь | Зависимость | Задача зависит или от неё зависят | Карточка, зависимости |
| ↩️ Стрелка вверх | Откат | Инженер откатил задачу из "На проверке" | История статусов |

## ПРИНЦИПЫ ДИЗАЙНА

1. **Статус всегда видна** — на карточке, в списке, в канбане
2. **Цвета консистентны** — красный = блокировка/срочно, зелёный = OK
3. **Одна кнопка = одно действие** — нет многозначности
4. **История сохранена** — вся хронология в "ИСТОРИИ СТАТУСОВ"
5. **Замечания структурированы** — критичность, место, тег, статус разрешения
6. **Уведомления срочные** — Telegram для критичного, IN-APP для всего
7. **Зависимости видны** — инженер знает почему задача блокирована
8. **Роль определяет UI** — инженер не видит "Очередь ГИПа"
