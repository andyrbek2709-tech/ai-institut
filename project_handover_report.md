# Технический отчет: EngHub AI Orchestrator (Handover Document)

Данный документ предназначен для полной передачи контекста стороннему разработчику. Он описывает архитектуру, текущее состояние и планы развития платформы для инжинирингового менеджмента.

## 1. Обзор проекта
**EngHub** — это платформа для управления проектными работами в промышленном проектировании с упором на автоматизацию, расчеты и AI-ассистирование (RAG).

### Стек технологий:
- **Frontend**: React (Vite / TypeScript / Vanilla CSS).
- **Backend / DB**: [Supabase](https://jbdljdwlfimvmqybzynv.supabase.co) (Auth, PostgREST, RLS).
- **Deployment**: [Vercel](https://enghub-jv6v27w66-andyrbek2709-techs-projects.vercel.app).
- **Core logic**: Прямые `fetch` запросы к REST API Supabase (без SDK для легковесности).

## 4. Архитектура и База Данных
Бэкенд реализован на Supabase с использованием Row Level Security (RLS). Основные таблицы:
- `projects`: Реестр проектов (сроки, статусы, закрепленные отделы).
- `tasks`: Задачи и межотдельские задания. Поддерживают **ревизионирование** (поле `revision_num`, `parent_task_id`).
- `app_users`: Профили пользователей (roles: `gip`, `lead`, `engineer`, `admin`).
- `departments`: Справочник отделов.
- `messages`: Чат внутри задач (`task_id`) и системные уведомления (тип `call_start`).
- `normative_docs`: (Phase 7) Метаданные документов для RAG-системы.

### Ролевая модель:
- **GIP (ГИП)**: Управление проектами, назначение Lead'ам, выпуск новых ревизий (R0 -> R1).
- **Lead (Нач. отдела)**: Назначение инженерам, приемка заданий от смежников, проверка работ.
- **Engineer**: Выполнение задач, ведение расчетов.
- **Admin**: Управление пользователями и структурой компании.

## 3. Текущий статус реализации

### ✅ Реализовано (Phases 1-3, 7):
1.  **Dashboard & Projects**: Полноценный UI для мониторинга прогресса, поиска по проектам и задачам.
2.  **Задания смежникам**: Матрица увязки (GIP View) и система Acceptance/Reject для межотдельских заданий.
3.  **Система ревизий**: Механизм клонирования задачи при выпуске ревизии R1/R2 с сохранением истории и замечаний.
4.  **Модуль расчетов**: Движок на JSON-шаблонах (Материал. баланс, Теплопотери, Кабели и т.д.) с экспортом в DOCX.
5.  **Нормативка (RAG UI)**: Раздел для загрузки PDF/DOCX, пакетная загрузка, фильтрация прав (загрузка только для ГИП).
6.  **AI Copilot**: Интегрированная панель чата с переключателем режима поиска по базе знаний (передает флаг `use_rag` на бэкенд).

### ⏳ В процессе (Критическое):
- **Vercel Build Error**: На текущий момент (31.03.2026) наблюдается сбой кэша сборки на Vercel (`react-scripts: command not found`). Проблема решается форсированной переустановкой зависимостей (`package.json` trigger).
- **RAG Backend**: UI готов, эмбеддинги и векторизация (pgvector) требуют настройки Edge Functions в Supabase или внешнего n8n-воркфлоу.

## 7. Где искать код (Key Code Locations)

- `src/App.tsx`: Основной файл приложения. Содержит роутинг (`screen`), стейты таблиц, логику навигации и рендеринг всех экранов (Dashboard, Projects, Tasks, Normative).
- `src/api/supabase.ts`: Хелперы для работы с API. **Важно**: Все запросы к БД идут через `post()`, `get()`, `patch()`.
- `src/components/CopilotPanel.tsx`: Логика ИИ-ассистента. Обрабатывает отправку сообщений и флаги RAG.
- `src/calculations/`: Папка с логикой инженерных расчетов.
- `upload_docs.js`: Технический скрипт для прямой инъекции документов в базу (использовался для тестов).

## 8. Как продолжить работу (Инструкция)
1. Склонировать репозиторий `enghub`.
2. Выполнить `npm install`.
3. Запустить проект через `npm run dev`.
4. Для деплоя на Vercel достаточно сделать `git push` в ветку `main` репозитория `enghub`.
