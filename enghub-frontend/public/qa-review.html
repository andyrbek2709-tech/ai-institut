Триггер task_history сломан - задачи не создаются

column "field" of relation "task_history" does not exist - на любом INSERT в tasks.
Не сделаноЛогика БДКорень найден

Корневая причина (подтверждено SQL)

На таблице public.tasks висят ТРИ триггера. Один корректный (использует field_name), два сломанных (используют field + comment, которых в таблице нет):

Колонки task_history: id, task_id, changed_by, field_name, old_value, new_value, changed_at - нет ни "field", ни "comment" Триггер task_changes_trigger (UPDATE) - OK: INSERT INTO task_history(task_id, field_name, old_value, new_value)... Триггер trg_tasks_history (UPDATE) - СЛОМАН: INSERT INTO task_history(task_id, field, old_value, new_value, changed_by, comment)... Триггер trg_tasks_history_create (INSERT) - СЛОМАН: INSERT INTO task_history(task_id, field, old_value, new_value, changed_by) VALUES (new.id, 'created', null, new.status, who); - срабатывает на КАЖДЫЙ INSERT - блокирует создание любой задачи

Решение (1 SQL-команда)

DROP TRIGGER IF EXISTS trg_tasks_history ON public.tasks; DROP TRIGGER IF EXISTS trg_tasks_history_create ON public.tasks; -- task_changes_trigger остаётся, он логирует UPDATE корректно.

Время фикса: 30 секунд через Supabase MCP. Можно прямо сейчас.
