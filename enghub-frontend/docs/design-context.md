## 1. Что такое EngHub

EngHub — это **цифровая система проектирования** для инженерных проектных институтов (стройка, нефтегаз, инфраструктура). Заменяет связку «Excel + Bitrix + Outlook» одним приложением.

**Кто пользуется:**

| Роль | Что делает | Сколько в системе |
|---|---|---|
| **Admin** | Управление пользователями, отделами | 1 |
| **GIP** (Главный инженер проекта) | Ведёт проект целиком: задачи, чертежи, координация отделов, согласование | 4 (планируется до 10) |
| **Lead** (Руководитель отдела) | Распределяет задачи внутри отдела, согласует первым уровнем | по 1 на отдел (8 отделов) |
| **Engineer** | Выполняет задачи, выпускает чертежи, отвечает на замечания | 5–50 на отдел |

**Ключевая особенность:** мы не TODO-приложение, а **система документооборота**. Чертёж → ревизия → трансмиттал → замечание — это технологическая цепочка, а не «список задач».

## 2. Технологический стек

| Слой | Технология |
|---|---|
| Фронт | React 18 + TypeScript |
| Стили | inline styles + минимальный CSS (нет Tailwind, нет CSS-in-JS библиотек) |
| Серверные функции | Vercel serverless (`api/*.js`) |
| БД | Supabase (PostgreSQL + RLS + Realtime) |
| Auth | Supabase Auth (email + JWT) |
| Видео | LiveKit Cloud |
| AI | OpenAI + Anthropic Claude (через `/api/orchestrator`) |
| Хостинг | Vercel (auto-deploy from `main`) |

> Дизайнеру важно: стили — **inline через объект `C` (цвета) и `getInp(C)` (инпуты)**. Если делаем новый компонент — наследуемся от существующих токенов, не создаём CSS-классы с нуля.

- Все акценты (accent/green/red/blue/purple/orange) — те же.

### Типографика

В коде нет явных `--font-family` переменных — используется системный стек. Размеры (наблюдательно):

- `page-title` — 28px (заголовок страницы)
- Заголовки карточек — 18–22px
- Текст таблиц / форм — 14px
- Подсказки / labels — 12px
- Метки / badge — 11px

Для дизайнера: **есть тренд переходить на Inter / SF Pro Display** на гладких корпоративных интерфейсах. Если предлагаешь — это допустимо. Сейчас фактически — что отдаст браузер.

## 4. Карта экранов

### Глобальная навигация (сайдбар)

```
⬡ Обзор          → Dashboard (виджеты + KPI + список проектов и задач)
◈ Проекты         → Реестр проектов (таблица + поиск + фильтр)
≡ Задачи          → Все задачи во всех проектах (для меня)
⎍ Расчёты         → Каталог инженерных расчётов (90 формул)
📋 Спецификации   → Спецификации оборудования (привязка к проекту)
📄 Нормативка     → ГОСТы, СНиПы, RAG-поиск по документам
```

Плюс в шапке:
- Глобальный поиск (по проектам/задачам/чертежам/замечаниям/нормативке)
- Уведомления (`NotificationCenter`)
- AI Copilot (выезжающая панель справа)
- Переключатель темы
- Профиль / выход

### Карточка проекта (вкладки)

```
🗣 Совещание   — видеовстреча (LiveKit)
⊙ Задачи        — таблица + Kanban с drag-and-drop
📐 Чертежи      — реестр чертежей (АР-101, КМ-12-ГП и т.д.)
🧾 Ревизии      — история ревизий каждого чертежа (R0 → R1 → R2)
📝 Замечания    — обсуждения с тредами комментариев
📦 Трансмитталы — выпуск пакетов (TR-ЖК-2026-01-001)
✉ Увязка       — координация заданий между смежниками
📊 Диаграмма    — Gantt
🗺 Timeline     — хронологическая лента событий проекта
🗒 Протоколы    — протоколы совещаний
⏱ Табель        — учёт времени
🏛 ГИП           — отдельный дашборд для ГИПа (только ему виден)
🏗 BIM          — placeholder (для GIP/Lead)
```

**Важно:** табы прокручиваются горизонтально на узких экранах (`overflow-x: auto`). Это место — больной мобильный UX.

### Админка (`/admin`)

```
Пользователи  — список + создание/редактирование (Modal)
Отделы        — справочник
```

### Login

Простая форма email + password + ссылка «забыли пароль». Тёмная тема по умолчанию.

## 6. Болевые точки UX (из QA-обзора 27.04.2026)

Это то, что **дизайнер должен решить** — не разработчик. Раскладываю по категориям.

### Логика модели

- **«Где в чертеже это замечание?»** — в карточке замечания нет поля «лист / узел / ось». В реальной практике инженер пишет «Узел А-3, лист 12, ось Б/4». Без этого замечание теряет адресность.
- **«Кому трансмиттал?»** — нет поля «получатель» и «организация-получатель». В реальной практике трансмиттал всегда адресован: заказчику, экспертизе, смежникам.

### Дашборд

- **«Что нового с моего последнего входа?»** — самый частый вопрос. Сейчас нет ленты активности. Нужен компонент типа feed.
- **Фильтры «Мои / Просроченные / На этой неделе»** — отсутствуют. Инженер чаще спрашивает «что делать сегодня», чем «как называется задача».

### Empty states

- Большинство пустых состояний — просто текст «Нет данных». Нет иллюстрации, нет CTA «Создать первую задачу». Новый пользователь теряется на первом экране.

### Мобильность

- **Сайдбар на мобиле** — проблема. Сейчас он всегда виден. Нужно: hamburger / drawer.
- **Табы карточки проекта** — горизонтальный скролл без визуального индикатора. Нет градиента/тени справа, чтобы дать понять «там ещё что-то».
- 13 табов в одной строке — слишком много. Возможно, нужна группировка («Документы», «Управление», «AI») с раскрытием.

### Микро-UX

- Длинные имена проектов в сайдбаре обрезаются (`text-overflow: ellipsis`) — теперь есть `title=` атрибут.
- Кнопки смены статуса задачи не предупреждают о необратимости перехода. Хорошо бы tooltip: «После отправки на проверку задача станет нередактируемой, пока её не вернут на доработку».
- Несколько мест ещё показывают US-формат даты `4/27/2026, 11:45:44 PM` — нужно `27.04.2026, 23:45`. (В коде это уже частично исправлено, но не везде.)
- Локализация статусов в тостах: «Статус трансмиттала изменён: issued» → должно быть «…изменён: Выпущен».

### Стиль

- Тёмная тема — **проработана хорошо**, контраст соответствует.
- Иконки — Unicode-символы (⬡ ◈ ≡ ⎍ 📋 📄). Это сознательное упрощение, но **дизайнер может предложить заменить на профессиональный icon set** (Phosphor / Lucide / Tabler) — вопрос только в том, тащить ли зависимость.
- Логотипа нет. Только название «EngHub» текстом.

## 7. Что НЕ нужно трогать

- **Цветовые токены (`constants.ts`)** — менять можно только всю палитру целиком, согласовав с владельцем. Нельзя добавлять hardcoded цвета в компоненты.
- **Сайдбар тёмный в обеих темах** — это сознательное решение (Stitch-aligned), не баг.
- **Структура «вкладок проекта»** — это терминология предметной области, дизайн менять можно, но названия и порядок (Чертежи → Ревизии → Замечания → Трансмитталы) — это технологическая цепочка, нельзя переставлять или переименовывать.

## 8. На что фокусироваться сейчас (топ-3)

1. **Мобильная навигация** — сайдбар → drawer, табы карточки проекта → группированные / горизонтальный скролл с индикатором.
2. **Лента активности на дашборде** — «что нового с моего последнего входа» (одна история на дашборде + аудиторская история на карточке проекта).
3. **Адресность замечаний и трансмитталов** — поля «место в чертеже» и «получатель». Это про UX модели, не CSS.

Дальше — empty states, микро-UX, иконки.
